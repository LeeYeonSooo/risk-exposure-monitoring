# 알림 taxonomy 최적화 + 적대적 검증 (같은 날 후속)

## taxonomy 검토 발견
- **중복(같은 kind, 다른 디텍터)**: bad_debt_threshold(diff 실현deficit vs scan 청산거리), high_utilization(diff triplet vs scan 고정warning vs nonevm), curator_derisk(snapshot-curators vs scan-curator-derisk 데드).
- **유령 kind**(프론트만 참조, 백엔드 미발화): total_supply_jump(실제론 supply_spike+supply_single_mint), supply_jump(이미 정리됨), flow_anomaly(데드 컴포넌트 LandingGraph에서만 참조) — 모두 탐지공백 아님, 표시 드리프트.
- **스케줄 안 됨**: scan-current-risks가 cron 미등록 → EVM reserve_frozen·high_lltv·unverified·aggLTV bad_debt 라이브 미생성.
- **severity 부적절**: collateral_adoption isMajor→무조건 critical, reserve_frozen 항상 warning.

## 적용
- collateral_adoption: isMajor 무조건 critical → **VERIFIED_CORE_COLLATERAL allowlist** 기반 fail-safe(코어만 info).
- high_utilization: scan 고정warning → diff와 동일 triplet 통일.
- bad_debt 분리: scan 근청산 신호 → **near_liquidation** kind(실현 deficit과 구분). DB active 3건 마이그레이션.
- 렌딩 경색 dedup: high_util(warning+) 발화 시 warning liquidity_drop만 info 강등(critical 보존).
- curator: snapshot-curators에 **신호3 완전회수(prev 역순회)** 추가 + scan-curator-derisk deprecated.
- 프론트: orphan 라벨(total_supply_jump) 제거, bad_debt/near_liq 카테고리 디페깅→유동성, near_liquidation 라벨 추가.
